String

Additional utility methods. A large number of these methods facilitate generation of CSS selectors based upon string variables. String manipulation in Javascript is ugly, and these methods are intended to reduce the boilerplate required when manipulating CSS selectors. For example, it is neater to write 
    x.cssID()
in place of 
    '#' + x
Each of these CSS selector methods has a css prefix. As they each return a string value, they are chainable, enabling the construction of complex selectors in highly-readable form.

cssAttr(value, comparison) → {string}

Constructs an attribute selector from the string. For example: 
    'selector1'.cssAttr() -> '[selector1]'
    'selector1'.cssAttr('value') -> '[selector1="value"]'
    'selector1'.cssAttr('value', 'start') -> '[selector1^="value"]'
Parameters:
Name Type Argument Default Description
value string <optional>
 The attribute value. Simple values only allowed.
comparison string <optional>
 'equal' The type of comparison. Recognised values are:
  • 'end' - $=
  • 'equal' - =
  • 'exact' - ~=
  • 'prefix' - |=
  • 'start' - ^=
